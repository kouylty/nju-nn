# 神经网络编程框架实验报告

## 一、实验主要内容

本次大作业的目标是实现一个简单的神经网络编程框架。框架需要覆盖神经网络训练流程中的主要组成部分，包括数据读取与批量加载、张量运算、自动微分、模块化网络层、参数初始化、优化器、学习率调度器、模型训练测试流程以及模型参数的保存和读取。

结合课程 PDF 和项目代码结构，本实验主要完成以下内容：

1. 实现 `Dataset` 和 `DataLoader`，支持从表格数据和图像数据中读取样本，并按 mini-batch 形式提供给训练过程。数据集目标包括 Iris、MNIST 和 Fashion-MNIST。
2. 实现张量计算和自动微分的核心机制，包括计算图节点 `Value`、张量 `Tensor`、计算操作 `Op` / `TensorOp`、拓扑排序和反向传播。
3. 实现常用张量算子，例如逐元素加减乘除、矩阵乘法、转置、变形、广播、求和、对数、指数和 ReLU 等。
4. 实现模块化神经网络组件，包括 `Module`、`Parameter`、`Linear`、`Flatten`、`ReLU`、`Sequential`、`Residual` 以及常用损失函数。
5. 实现 Xavier 和 He 参数初始化方法。
6. 实现优化器和学习率规划器，包括 SGD、Momentum、Adam、weight decay，以及 StepDecay、LinearWarmUp、CosineDecayWithWarmRestarts 等学习率调度策略。
7. 使用该简易框架完成 ResidualMLP 实验，并实现训练、测试、模型保存和模型读取。测试阶段应避免构建无用梯度。

## 三、自动微分核心实现

本阶段完成的是实验计划中的第二步，即实现自动微分的核心流程。修改位置为 `python/needle/autograd.py` 中的两个函数：

- `find_topo_sort(node_list)`
- `compute_gradient_of_variables(output_tensor, out_grad)`

### 1. 拓扑排序

计算图中的每个 `Value` / `Tensor` 节点只保存自己的输入节点 `inputs`，并不保存输出节点。因此拓扑排序需要从输出节点出发，沿着 `inputs` 反向遍历整张计算图。

实现时采用深度优先搜索的后序遍历：

1. 从目标输出节点开始 DFS。
2. 对当前节点，先递归访问它的所有输入节点。
3. 输入节点都访问完后，再把当前节点加入 `topo_order`。
4. 使用 `visited` 集合避免重复访问共享子图中的节点。

这样得到的拓扑序满足：一个节点的输入一定排在该节点之前。也就是说，前向依赖关系是从前到后排列的。反向传播时只需要将这个拓扑序反转，就可以从输出节点逐步回传到叶子节点。

### 2. 反向传播

反向传播函数 `compute_gradient_of_variables` 的核心思想是维护一个字典：

```python
node_to_output_grads_list: Dict[Tensor, List[Tensor]]
```

该字典记录每个节点从所有输出边收到的局部伴随值，也就是 partial adjoint。由于同一个中间节点可能被多个后续节点使用，因此它可能收到多份梯度贡献。反向传播时需要先把这些梯度贡献求和，得到该节点最终的 `node.grad`。

具体流程如下：

1. 先将输出节点的初始梯度设为 `out_grad`。
2. 对输出节点做拓扑排序，并反转拓扑序。
3. 按反向拓扑序遍历每个节点。
4. 将当前节点收到的所有梯度贡献用 `sum_node_list` 相加，保存到 `node.grad`。
5. 如果当前节点是叶子节点，则没有前驱操作，不继续传播。
6. 如果当前节点不是叶子节点，则调用 `node.op.gradient_as_tuple(node.grad, node)`，计算该操作对每个输入节点的梯度贡献。
7. 将每个输入节点收到的新梯度追加到 `node_to_output_grads_list` 中。

这一实现与课程 PDF 中给出的自动求导伪代码一致：先按反向拓扑序求当前节点的总伴随值，再通过当前节点对应的局部梯度规则，把梯度继续传回输入节点。

### 3. 当前实现的意义

完成这一步后，框架具备了自动微分的基本调度能力。后续只要每个 `TensorOp` 正确实现自己的 `gradient` 方法，整个计算图就可以自动完成链式求导。例如，矩阵乘法、广播、求和、ReLU 和交叉熵损失等算子的梯度规则补齐后，就可以支持 `loss.backward()`，并让优化器读取每个参数的 `param.grad` 来更新网络参数。

本阶段只实现自动微分的图遍历和梯度传播框架，并未实现具体算子的梯度公式。具体 TensorOp 的前向计算和局部梯度将在后续步骤中完成。

## 四、基础张量算子实现

本阶段完成的是实验计划中的第三步，即补齐 `ops_mathematic.py` 和 `ops_logarithmic.py` 中的基础张量算子。该步骤的目标是让前一步实现的自动微分框架真正可以对神经网络中的常用运算进行前向计算和反向求导。

本次实现的算子包括：

- `PowerScalar`
- `EWiseDiv`
- `DivScalar`
- `Transpose`
- `Reshape`
- `BroadcastTo`
- `Summation`
- `MatMul`
- `Negate`
- `Log`
- `Exp`
- `ReLU`
- `Tanh`
- `LogSoftmax`
- `LogSumExp`

此外，还修正了 `EWisePow` 的实现。原始代码中 `EWisePow.gradient` 判断输入类型的逻辑不正确，因为计算图中 `node.inputs` 保存的是 `Tensor` 节点，而不是底层的 `NDArray` 数据。同时，`needle` 后端中底层 `NDArray.__pow__` 路径并不稳定，因此对 `EWisePow.compute` 和 `PowerScalar.compute` 分别处理了 NumPy 后端和 needle 后端。

### 1. 基础逐元素算子

逐元素算子包括幂、除法、取负、对数、指数、ReLU 和 Tanh。这些算子的前向计算直接调用后端提供的数组计算接口，反向传播根据基本求导公式实现。

主要梯度公式如下：

```text
d(a ** c) / da = c * a ** (c - 1)
d(a / b) / da = 1 / b
d(a / b) / db = -a / b^2
d(log a) / da = 1 / a
d(exp a) / da = exp(a)
d(tanh a) / da = 1 - tanh(a)^2
```

对于 `ReLU`，前向计算为 `max(a, 0)`，反向传播时使用输入是否大于 0 的 mask：

```python
grad = out_grad * (a > 0)
```

这里的 mask 通过输入节点的 `cached_data` 构造为不需要梯度的 `Tensor`，再参与反向传播计算。

### 2. 形状变换算子

`Transpose` 和 `Reshape` 这类算子只改变张量的视图或形状，不改变元素的数值关系。

`Reshape` 的反向传播比较直接：前向从原形状变为目标形状，反向时只需要把上游梯度 reshape 回输入张量的原始形状。

`Transpose` 的反向传播是再次交换相同的轴。因为转置操作本身是一个置换，逆操作仍然可以用相同轴对的转置表示。对于未指定轴的情况，默认交换最后两个维度，与矩阵转置一致。

实现时还考虑了后端差异：`needle` 后端使用 `array_api.permute`，NumPy 后端使用 `np.transpose`。

### 3. 广播和求和

`BroadcastTo` 和 `Summation` 是一对在反向传播中经常互相出现的操作。

对于 `Summation`，前向会把某些维度消去。反向传播时，需要先把上游梯度 reshape 成带有被消去维度的形状，再 broadcast 回输入张量原始形状。例如，对形状 `(m, n)` 的张量按第 1 维求和后，输出形状为 `(m,)`，反向时需要先变成 `(m, 1)`，再广播为 `(m, n)`。

对于 `BroadcastTo`，前向会把某些长度为 1 的维度扩展到更大的尺寸，或者在前面补出新维度。反向传播时，需要把这些被广播出来的维度求和压回原来的形状。为此实现了辅助函数 `_sum_to_shape`，专门处理“将梯度规约回指定 shape”的逻辑。

这一部分是后续实现 bias 加法、BatchNorm、LayerNorm 和损失函数时必须依赖的基础。

### 4. 矩阵乘法

`MatMul` 的前向计算使用矩阵乘法。对于普通二维矩阵：

```text
C = A @ B
```

其梯度公式为：

```text
dL/dA = dL/dC @ B^T
dL/dB = A^T @ dL/dC
```

实现中还兼容了 batched matmul。当输入存在 batch 维度或发生广播时，反向传播得到的梯度形状可能大于原输入形状，因此同样使用 `_sum_to_shape` 把梯度规约回原始输入 shape。

这使得该算子既能支持普通全连接层中的二维矩阵乘法，也可以支持更高维张量中的批量矩阵乘法。

### 5. LogSoftmax 和 LogSumExp

`LogSoftmax` 和 `LogSumExp` 是分类损失函数中非常重要的数值稳定算子。为了避免直接计算 `exp` 时溢出，`LogSumExp` 使用稳定形式：

```text
logsumexp(Z) = max(Z) + log(sum(exp(Z - max(Z))))
```

`LogSoftmax` 的前向计算可以写成：

```text
logsoftmax(Z) = Z - logsumexp(Z)
```

其中 `logsumexp` 默认沿最后一个维度计算，并广播回原始形状。

`LogSumExp` 的梯度本质上是 softmax 权重：

```text
d logsumexp(Z) / dZ = exp(Z - logsumexp(Z))
```

`LogSoftmax` 的梯度为：

```text
dL/dZ = out_grad - softmax(Z) * sum(out_grad)
```

实现时还补充了多轴 reduction 的 shape 处理，包括 `axes=None`、负轴、单轴和多轴。由于底层 `NDArray.sum/max` 对多轴规约支持有限，代码在 op 层按轴从大到小逐个规约，并通过 `_keepdims_shape` 和 `_remove_axes_shape` 恢复需要的形状。

### 6. 检查结果

完成实现后，对两个算子文件进行了语法检查，并确认 `ops_mathematic.py` 和 `ops_logarithmic.py` 中已经没有剩余的 `raise NotImplementedError()`。

随后进行了若干小规模功能测试：

1. 在 `needle` 后端下测试矩阵乘法、`logsumexp`、`logsoftmax` 的组合前向与反向传播，梯度能够正常计算。
2. 在 NumPy 后端下测试同样的组合算子，前向和反向传播结果正常。
3. 测试 batched matmul，输出形状和两个输入的梯度形状都符合预期。
4. 测试 `logsumexp` 的 `axes=None`、`axes=0`、`axes=1`、`axes=(0, 1)` 和 `axes=-1`，输出形状和数值正常。
5. 对 `power`、`log`、`exp`、`tanh` 的组合函数与 NumPy 解析梯度进行比较，最大梯度误差为 0。

这些检查说明基础张量算子已经可以接入自动微分框架，为后续实现神经网络模块、损失函数和优化器提供了必要的运算基础。

## 五、初始化方法与基础神经网络模块实现

本次实现或补充的神经网络模块包括：

- `Sequential`

- `Softmax`
- `SoftmaxLoss`
- `CrossEntrophyLoss`

- `MSELoss`
- `BatchNorm1d`
- `LayerNorm1d`

- `Residual`

其中 `CrossEntrophyLoss` 和 `BinaryCrossEntrophyLoss` 保留了原始代码中的拼写，另外新增了正确拼写的别名 `CrossEntropyLoss` 和 `BinaryCrossEntropyLoss`，方便后续实验脚本使用。

### 1. 参数初始化

参数初始化函数位于 `python/needle/init/init_initializers.py`。本实验实现了 Xavier 和 Kaiming/He 两类初始化方法。

Xavier 初始化主要适用于 Sigmoid、Tanh 等激活函数，目标是让前向传播和反向传播中的方差尽量稳定。其均匀分布形式为：

```text
W ~ U(-a, a)
a = gain * sqrt(6 / (fan_in + fan_out))
```

正态分布形式为：

```text
W ~ N(0, std^2)
std = gain * sqrt(2 / (fan_in + fan_out))
```

Kaiming/He 初始化主要适用于 ReLU 激活函数。其均匀分布形式为：

```text
W ~ U(-a, a)
a = sqrt(6 / fan_in)
```

正态分布形式为：

```text
W ~ N(0, std^2)
std = sqrt(2 / fan_in)
```

实现时，如果用户没有显式传入 `shape`，默认使用 `(fan_in, fan_out)`，可以直接用于全连接层的权重初始化。同时，为了避免参数和输入张量落在不同后端设备上，初始化函数在没有显式传入 `device` 时使用 `default_device()`。

### 2. 顺序模型与残差连接

`Sequential` 用于把多个模块顺序组合起来。其前向传播逻辑是依次调用内部保存的每个子模块：

```python
for module in self.modules:
    x = module(x)
return x
```

这样可以用较简洁的方式定义多层神经网络，例如 `Linear -> ReLU -> Linear`。

`Residual` 用于实现残差连接。其前向传播为：

```python
return self.fn(x) + x
```

即先将输入传入子模块 `fn`，再把子模块输出与原输入相加。该模块是后续实现 ResidualMLP 的核心组件。

### 3. 激活函数与 Softmax

`ReLU` 已经通过 `ops.relu` 接入张量算子。本阶段新增了 `Sigmoid` 和 `Softmax`。

`Sigmoid` 使用指数函数实现：

```text
sigmoid(x) = 1 / (1 + exp(-x))
```

由于当前框架中的 `Tensor.__rsub__` 对 `1 - tensor` 这类表达式支持不完善，实现时尽量使用不会触发该问题的等价写法。

`Softmax` 使用前一阶段实现的稳定 `logsoftmax`：

```text
softmax(x) = exp(logsoftmax(x))
```

这样可以复用 `LogSoftmax` 中的数值稳定处理。

### 4. 损失函数

`SoftmaxLoss` 用于多分类任务，输入为 logits 和类别标签。实现时没有直接计算 softmax，而是使用稳定的 log-sum-exp 形式：

```text
loss = logsumexp(logits) - logits[y]
```

在代码中，先将类别标签转为 one-hot，再计算每个样本的交叉熵，最后对 batch 求平均。

`CrossEntrophyLoss` 用于标签已经是 one-hot 或概率分布的情况。它先计算：

```text
log_probs = logsoftmax(logits)
```

然后计算：

```text
loss = -sum(y * log_probs) / batch_size
```

`BinaryCrossEntrophyLoss` 用于二分类概率输入，公式为：

```text
loss = -[y * log(p) + (1 - y) * log(1 - p)]
```

实现时加入很小的 `eps`，避免出现 `log(0)`。同时为了绕开当前 `Tensor.__rsub__` 的限制，代码中使用 `-y + 1` 和 `-p + 1` 表示 `1 - y` 和 `1 - p`。

`MSELoss` 用于回归任务，计算预测值和目标值之间的均方误差：

```text
loss = mean((input - target)^2)
```

### 5. BatchNorm1d

`BatchNorm1d` 对二维输入 `(batch_size, dim)` 的每个特征维度进行归一化。训练阶段先计算 batch 均值和方差：

```text
mean = sum(x, axis=0) / batch_size
var = sum((x - mean)^2, axis=0) / batch_size
```

然后进行归一化和仿射变换：

```text
y = (x - mean) / sqrt(var + eps) * weight + bias
```

其中 `weight` 和 `bias` 是可训练参数，形状为 `(dim,)`。

训练阶段还会使用 momentum 更新 `running_mean` 和 `running_var`：

```text
running = (1 - momentum) * running + momentum * batch_stat
```

测试阶段则不再使用当前 batch 的统计量，而是使用训练过程中累计的 running mean 和 running var。

实现时需要显式 reshape 和 broadcast 均值、方差、权重和偏置，因为当前框架的逐元素加减乘除不会自动完成所有广播。

### 6. LayerNorm1d

`LayerNorm1d` 对每个样本的最后一维进行归一化。与 BatchNorm 不同，LayerNorm 不依赖 batch 统计量，而是在每个样本内部计算均值和方差：

```text
mean = mean(x, last_dim)
var = mean((x - mean)^2, last_dim)
```

随后同样进行归一化和仿射变换。为了保留维度，代码中实现了 `_sum_keepdims` 和 `_mean_keepdims` 辅助函数，先沿指定轴求和，再 reshape 成 keepdims 的形式，便于后续 broadcast。

### 7. Dropout

`Dropout` 在训练阶段随机将部分元素置零，并对剩余元素进行缩放：

```text
y = x * mask / (1 - p)
```

其中 `mask` 以概率 `1 - p` 取 1，以概率 `p` 取 0。这样可以保证训练阶段输出的期望与输入一致。

在 eval 模式下，`Dropout` 直接返回原输入，不再随机置零。

## 六、优化器实现

本阶段完成的是实验计划中的第五步，即补齐 `python/needle/optim.py` 中的优化器实现。优化器负责读取自动微分阶段写入到参数上的梯度，并根据指定更新规则修改模型参数。

本次实现的内容包括：

- `SGD.step`
- `SGD.clip_grad_norm`
- `Adam.step`

同时，为了避免优化器内部状态被意外接入计算图，本阶段还对优化器状态和参数更新过程做了显式 `detach` 处理。

### 1. Optimizer 基类

项目中的优化器都继承自 `Optimizer` 基类。基类保存参数列表：

```python
self.params = params
```

并提供 `reset_grad` 方法：

```python
for p in self.params:
    p.grad = None
```

一次标准训练迭代通常为：

```python
loss.backward()
optimizer.step()
optimizer.reset_grad()
```

其中 `loss.backward()` 负责计算梯度，`optimizer.step()` 负责更新参数，`optimizer.reset_grad()` 负责清空旧梯度，避免下一轮反向传播时梯度残留。

### 2. SGD 参数更新

`SGD.step` 遍历所有参数，并跳过 `grad is None` 的参数。普通 SGD 的更新公式为：

```text
param = param - lr * grad
```

如果设置了 weight decay，则使用 L2 正则化形式修正梯度：

```text
grad = grad + weight_decay * param
```

如果设置了 momentum，则使用速度缓存 `u`：

```text
u = momentum * u + grad
param = param - lr * u
```

代码中使用参数对象本身作为字典 key：

```python
self.u[param]
```

用于保存每个参数独立的动量状态。

这里采用的是常见深度学习框架中的 SGD momentum 写法，即 `u = momentum * u + grad`。另一种写法 `u = momentum * u + (1 - momentum) * grad` 更接近指数滑动平均，会改变学习率的实际尺度。本实验采用前者，使其行为更接近常见的 SGD momentum。

### 3. 梯度裁剪

`SGD.clip_grad_norm` 用于在参数更新前限制梯度整体大小，避免梯度爆炸。

首先计算所有参数梯度的全局 L2 范数：

```text
total_norm = sqrt(sum(grad^2))
```

如果该范数超过给定阈值 `max_norm`，就对所有参数梯度乘以同一个缩放系数：

```text
scale = max_norm / (total_norm + 1e-6)
grad = grad * scale
```

这样可以保持所有参数梯度的相对方向不变，只缩小整体长度。函数返回裁剪前的 `total_norm`，便于后续训练时记录日志或分析训练稳定性。

### 4. Adam 参数更新

Adam 使用一阶矩和二阶矩对梯度进行自适应缩放。每次调用 `step` 时，先将时间步 `t` 加一：

```python
self.t += 1
```

然后对每个参数维护两个状态：

```text
u: 梯度一阶矩
v: 梯度二阶矩
```

其更新公式为：

```text
u = beta1 * u + (1 - beta1) * grad
v = beta2 * v + (1 - beta2) * grad^2
```

由于初始时 `u` 和 `v` 为 0，Adam 需要进行 bias correction：

```text
u_hat = u / (1 - beta1^t)
v_hat = v / (1 - beta2^t)
```

最终参数更新为：

```text
param = param - lr * u_hat / (sqrt(v_hat) + eps)
```

如果设置了 weight decay，则与 SGD 一样，先对梯度加入 L2 正则项：

```text
grad = grad + weight_decay * param
```

### 5. detach 处理

优化器的状态不应该成为神经网络计算图的一部分。否则随着训练迭代增加，优化器状态可能保留旧计算图，导致不必要的内存占用和错误的梯度依赖。

因此本阶段对以下数据显式调用了 `detach()`：

1. 进入优化器更新逻辑的 `param.grad`。
2. SGD 的 momentum 缓存 `self.u[param]`。
3. 梯度裁剪后写回的 `param.grad`。
4. Adam 的一阶矩 `self.u[param]`。
5. Adam 的二阶矩 `self.v[param]`。
6. Adam 中用于参数更新的 `update`。
7. 写回参数的 `param.data`。

例如，SGD momentum 缓存更新后会执行：

```python
self.u[param] = self.u[param].detach()
```

Adam 中的一阶矩和二阶矩更新后也会执行：

```python
self.u[param] = self.u[param].detach()
self.v[param] = self.v[param].detach()
```

这样可以保证优化器只保存数值状态，而不保存训练计算图。

### 6. 检查结果

完成实现后，对 `optim.py` 进行了语法检查。随后构造小型网络，分别测试了普通 SGD、带 momentum 和 weight decay 的 SGD，以及 Adam。

测试流程为：

1. 构造一个小型 `Sequential` 网络。
2. 前向计算 `MSELoss`。
3. 调用 `loss.backward()` 计算梯度。
4. 对 SGD 调用 `clip_grad_norm`。
5. 调用 `optimizer.step()` 更新参数。
6. 调用 `optimizer.reset_grad()` 清空梯度。

检查结果表明：

1. `SGD`、带 momentum/weight decay 的 `SGD` 和 `Adam` 都能完成一次参数更新。
2. 更新后参数数值确实发生变化。
3. `reset_grad` 后所有参数梯度被清空。
4. 默认 needle 后端和 NumPy 后端的小模型测试均通过。

这些检查说明优化器已经可以和前面实现的自动微分、张量算子、神经网络模块配合使用，后续可以接入完整训练循环。

### 2. 数据预处理脚本

数据预处理脚本为 `data/prepare_data.py`。该脚本只负责从本地 `data/raw` 中读取数据，不负责联网下载。

`--image-shape flat` 表示将 MNIST 和 Fashion-MNIST 图像保存为展平后的 `(N, 784)`。如果使用 `--image-shape image`，则保存为 `(N, 28, 28)`。

脚本对 MNIST/Fashion-MNIST 的处理流程如下：

1. 使用 `gzip` 读取 `.gz` 文件。

4. 使用分层划分，按类别保持比例划分训练集和测试集。
5. 默认测试集比例为 0.2，即 120 条训练样本、30 条测试样本。
6. 保存为 `.npz` 文件。

### 3. ResidualMLP 模型结构

ResidualMLP 模型定义在 `apps/residual_mlp.py`。该模型用于表格数据或展平后的图像数据。输入为：

```text
(batch_size, input_dim)
```

输出为：

```text
(batch_size, num_classes)
```

整体结构为：

```text
input
 -> Linear(input_dim, hidden_dim)
 -> ReLU
 -> ResidualMLPBlock * num_blocks
 -> Linear(hidden_dim, num_classes)
```

每个残差块 `ResidualMLPBlock` 的结构为：

```text
x
 -> Residual(
      Linear(hidden_dim, hidden_dim)
      ReLU
      Linear(hidden_dim, hidden_dim)
    )
 -> ReLU
```

也就是：

```text
block(x) = ReLU(x + Linear(ReLU(Linear(x))))
```

这种结构先通过输入层将原始特征映射到统一的 hidden 维度，然后在 hidden 空间中堆叠残差块。残差连接要求输入输出维度一致，因此残差块内部全部使用 `hidden_dim -> hidden_dim` 的线性层。

模型构造函数支持以下参数：

```python
ResidualMLP(
    input_dim,
    hidden_dim,
    num_classes,
    num_blocks=2,
    device=None,
    dtype="float32",
)
```

其中：

- `input_dim` 为输入维度，Iris 为 4，MNIST/Fashion-MNIST 为 784。
- `hidden_dim` 为隐藏层宽度。
- `num_classes` 为分类类别数。
- `num_blocks` 为残差块数量。

在 `forward` 中，如果输入维度大于 2，例如 `(N, 1, 28, 28)`，模型会自动展平成 `(N, 784)`，从而兼容图像格式输入和已经展平的输入。

### 4. 训练脚本实现

训练脚本为 `apps/train_residual_mlp.py`。由于当前 `DataLoader` 仍未实现，该脚本内部先实现了一个简单的 numpy batch 迭代器：

```python
batch_iter(X, y, batch_size, shuffle)
```

训练脚本支持以下功能：

1. 从 `data/processed/<dataset>/train.npz` 和 `test.npz` 加载数据。
2. 根据数据集自动设置输入维度和类别数。
3. 构造 `ResidualMLP`。
4. 选择优化器：`SGD` 或 `Adam`。
5. 使用 `SoftmaxLoss` 训练分类模型。
6. 每个 epoch 后计算训练集和测试集的 loss/accuracy。
7. 支持梯度裁剪。
8. 支持保存模型参数。
9. 支持加载模型参数。
10. 支持 eval-only 模式，只评估模型。

训练循环的核心流程为：

```python
logits = model(x_tensor)
loss = loss_fn(logits, y_tensor)
loss.backward()
optimizer.step()
optimizer.reset_grad()
```

测试阶段调用：

```python
model.eval()
```

并且只进行前向计算、loss 计算和 accuracy 统计，不调用 `backward()`，从而满足“测试阶段不计算梯度”的要求。

模型参数保存使用 `np.savez_compressed`，将 `model.parameters()` 中的每个参数按顺序保存为：

```text
param_0
param_1
...
```

加载模型时，按相同顺序读取参数，并写回每个 `param.data`。
